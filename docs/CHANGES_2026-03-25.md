# Session Changes — 2026-03-25

## Context

Analysis of adversarial test results and per-class F1 scores revealed that the model's persistent weakness on urgency classification (macro F1 ~0.62 across all v2 runs) was rooted in data quality rather than model architecture. The synthetic complaints generated by `gpt-4o-mini` in batches of 15 produced text where writing styles did not reliably match their intended labels — sarcastic-style complaints were labeled Low emotion, and Medium urgency complaints were linguistically indistinguishable from Low and High. No hyperparameter change (class weights, dropout, LR, label smoothing) closed the gap, pointing to noisy training signal as the root cause.

Two fixes were applied to `data_generation/generate_complaints.py`.

---

## Fix 1 — Model upgraded from `gpt-4o-mini` to `gpt-5-mini`

**Before:**
```python
MODEL = "gpt-4o-mini"
```

**After:**
```python
MODEL = "gpt-5-mini"
```

**Why:** `gpt-4o-mini` struggled to hold multiple simultaneous constraints (urgency level + emotion level + writing style + customer profile) faithfully across a batch of 15. The generated text tended to average out styles rather than committing to them — passive-aggressive samples felt flat, and High emotion did not reliably manifest as cold controlled anger when paired with formal styles. `gpt-5-mini` produces more consistent label-to-text fidelity at comparable cost (~$1.77 for 5,000 complaints vs ~$0.53 for `gpt-4o-mini`).

**Observed effect:** Urgency macro F1 jumped from 0.62 (best v2) to **0.80** on the v3 dataset. Medium urgency F1 specifically improved from 0.48 → 0.74. Qualitative inspection confirmed that sarcastic-style samples now read as genuinely sarcastic, and High/Low urgency-emotion divergent cases (e.g. calm description of a catastrophic outage) are more faithfully rendered.

---

## Fix 2 — Batch size reduced from 15 to 5

**Before:**
```python
BATCH_SIZE = 15
```

**After:**
```python
BATCH_SIZE = 5
```

**Why:** Requesting 15 complaints in a single API call caused quality degradation from context saturation. With 15 complaints to write simultaneously, the model converged on similar emotional registers by complaint 8–15, and writing styles lost their distinctiveness. Reducing to 5 per call gives the model focused attention per complaint, at the cost of more API calls (1,000 vs 334 for 5,000 complaints). The additional input token overhead from smaller batches adds ~$0.13 to the total cost.

---

## Net result

| Metric | v2 dataset (best run) | v3 dataset |
|---|---|---|
| Urgency macro F1 | 0.622 | **0.801** |
| Urgency Medium F1 | 0.480 | **0.736** |
| Emotion macro F1 | 0.868 | 0.852 |
| Adversarial test | 3 / 10 | **6 / 10** |

Emotion macro F1 decreased slightly (0.868 → 0.852) because the v3 dataset contains more nuanced emotional expression, making the task genuinely harder. The v2 emotion score was partly inflated by simpler, more formulaic text.
